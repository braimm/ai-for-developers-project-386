На этом шаге мы делаем UI приложения на основе контракта из предыдущего шага.
На этом шаге важно собрать рабочий интерфейс.

Требования:
1. Фронтенд реализуется как отдельная часть приложения.
2. Фронтенд получает данные и выполняет действия только через API по контракту.
3. Интерфейс должен корректно работать с отдельно запущенным бэкендом.
